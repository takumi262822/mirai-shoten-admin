# 画面説明: Mirai Shoten Admin (mirai-shoten-admin)

本画面説明の対象は 未来商店商材 に対応する管理画面である。実装の詳細設計は DESIGN.md に分離し、この文書では画面構成と操作導線に限定して説明する。

## 1. 目的
- 開発目的: 注文管理ダッシュボードを題材に、一覧確認、更新作業、集計確認を 1 画面で完結させる設計を示す。
- 評価してほしい点: ダッシュボードの見通し、CRUD 操作の分かりやすさ、利用者向け画面との役割分担。

## 2. 画面構成

### 2.1 ダッシュボード上部
- 対象: index.html のファーストビュー
- 主な要素:
  - タイトル
  - テーマ切替ボタン
  - 集計カード
- 役割: 画面に入った直後に、注文全体の状況を把握できるようにする。

### 2.2 注文フォーム
- 対象: ダッシュボード内の入力エリア
- 主な要素:
  - 顧客名
  - メールアドレス
  - 商品コード
  - 数量
  - 金額
  - ステータス
  - 追加 / 更新 / 編集キャンセル
- 役割: 新規登録と編集の入口を同じ場所にまとめ、運用時の視線移動を減らす。

### 2.3 注文一覧テーブル
- 対象: orderTableContainer
- 主な要素:
  - 注文 ID
  - 顧客情報
  - 商品コード、数量、金額
  - ステータス
  - 編集、削除ボタン
- 役割: 既存注文の確認と更新操作を一か所で行えるようにする。

### 2.4 顧客情報テーブル
- 対象: customerTableContainer
- 主な要素:
  - 顧客名
  - メールアドレス
  - 注文回数
  - 累計購入額
- 役割: 注文単位ではなく顧客単位でも状況を確認できるようにする。

## 3. 操作導線
- 起動後に集計カード、注文一覧、顧客一覧が表示される。
- 注文フォームから新規登録すると、一覧と集計が同じ画面内で更新される。
- 一覧の編集ボタンからフォームへ値が反映され、更新後に再描画される。
- 削除ボタンは確認ダイアログを経て反映される。
- テーマ切替で閲覧モードを変更できる。

## 4. 画面上の見どころ
- 上部で全体状況、中央で編集、下部で一覧確認という順に情報密度を調整している。
- 追加と編集を別画面にせず、同じフォームを使って運用作業を短くしている。
- 顧客一覧を独立表示することで、注文一覧だけでは見えにくい利用傾向も把握しやすい。
- 利用者向けの 未来商店商材 とは責務を分け、こちらは管理作業だけに集中した画面にしている。

== 概要

本書はプロジェクト「{project}」のシステム構成を記述した設計書です。
React + TypeScript（フロントエンド）/ Node.js + Express（バックエンド API）/ Supabase（PostgreSQL）の三層構成で構築します。

|JWT（jsonwebtoken + bcrypt）

== 通信フロー

=== ログイン処理

[source,mermaid]
....
sequenceDiagram
  participant C as ブラウザ (React)
  participant A as Express API
  participant D as Supabase DB

  C->>A: POST /api/auth/login { email, password }
  A->>D: SELECT id, password_hash FROM users WHERE email = ?
  D-->>A: ユーザーレコード
  A->>A: bcrypt.compare(password, hash)
  A-->>C: 200 OK { token: "eyJ..." }
  C->>C: localStorage.setItem("token", ...)
....

=== 認証が必要なリクエスト（注文一覧取得を例に）

[source,mermaid]
....
sequenceDiagram
  participant C as ブラウザ (React)
  participant A as Express API
  participant D as Supabase DB

  C->>A: GET /api/orders (Authorization: Bearer <token>)
  A->>A: jwt.verify(token, JWT_SECRET)
  A->>D: SELECT orders JOIN order_items
  D-->>A: レコード配列
  A-->>C: 200 OK { orders: [...] }
....

=== トークン失効時

[source,mermaid]
....
sequenceDiagram
  participant C as ブラウザ (React)
  participant A as Express API

  C->>A: GET /api/orders (期限切れ JWT)
  A->>A: jwt.verify() → TokenExpiredError
  A-->>C: 401 Unauthorized { error: "Token expired" }
  C->>C: localStorage.removeItem("token") → /login へリダイレクト
....

== 環境変数

=== バックエンド（.env）

[cols="2,1,5", options="header"]
|===
|変数名 |必須 |説明

|`DATABASE_URL`
|必須
|Supabase の PostgreSQL 接続文字列

|`JWT_SECRET`
|必須
|JWT 署名に使う秘密鍵（32文字以上推奨）

|`JWT_EXPIRES_IN`
|任意
|JWT 有効期限（例: `8h`）。デフォルト `8h`

|`PORT`
|任意
|サーバーのリスニングポート。デフォルト `3000`

|`CORS_ORIGIN`
|必須
|フロントエンドの URL（例: `https://your-app.vercel.app`）

|===

=== フロントエンド（.env）

[cols="2,1,5", options="header"]
|===
|変数名 |必須 |説明

|`VITE_API_URL`
|必須
|バックエンド API のベース URL（例: `https://your-api.onrender.com`）

|===

== セキュリティ方針

[cols="2,5", options="header"]
|===
|対策項目 |実装内容

|XSS
|フロントエンドで `escapeHtml()` 適用。React の JSX は基本的にエスケープ済み。

|SQL インジェクション
|Supabase クライアントのパラメータバインドを使用（生クエリ直接連結は禁止）

|認証バイパス
|全プロテクトルートに `authMiddleware` を適用。JWT 検証失敗時は 401 を返す

|パスワード保存
|bcrypt（salt rounds 12）でハッシュ化して保存。平文保存は禁止

|CORS
|`CORS_ORIGIN` 環境変数に登録した URL のみ許可。`*` は使用しない

|レートリミット
|ログインエンドポイントに `express-rate-limit` 適用（例: 10回/15分）

|===

== 概要

本書はプロジェクト「{project}」の主要コンポーネント・サービス・フックの構成と処理内容を記述した技術設計書です。

フロントエンドは React 18 + TypeScript + Vite、バックエンドは Node.js + Express + TypeScript、DB は Supabase（PostgreSQL）を使用します。

---

=== コンポーネント・フック設計

[cols="3,5", options="header"]
|===
|ファイル |担当責務

|`App.tsx`
|`BrowserRouter` でルーティングを設定。`/login` のみ公開ルート、残りは `PrivateRoute` で保護。

|`PrivateRoute.tsx`
|localStorage に JWT が存在しない場合 `/login` にリダイレクトする。認証設計書参照。

|`pages/Login.tsx`
|メール・パスワードでログインフォームを描画し `POST /api/auth/login` を呼び出す。成功時 JWT を保存して `/dashboard` へ遷移。

|`pages/Dashboard.tsx`
|`useStats` フックで統計カード（総注文数・売上・処理中・完了）を表示。直近5件の注文もリスト表示。

|`pages/Orders.tsx`
|`useOrders` フックで注文一覧を取得。ステータスフィルター・ページネーション対応。

|`pages/OrderDetail.tsx`
|URL の `:id` で注文を取得・表示。ステータス変更・全フィールド編集・削除が可能。

|`pages/OrderNew.tsx`
|注文追加フォーム。`POST /api/orders` で新規登録後 `/orders` にリダイレクト。

|`pages/Customers.tsx`
|`useCustomers` フックで顧客一覧を表示（注文回数・累計金額を集計済み）。

|`hooks/useOrders.ts`
|`GET /api/orders?status=&page=&limit=` を呼び出し `{ orders, total, loading, error }` を返す。

|`utils/api.ts`
|fetch ラッパー。JWT ヘッダー付与・401 レスポンス時の自動ログアウト・共通エラーハンドリングを提供。

|`utils/validator.ts`
|全フォームの入力バリデーション関数（email 形式・必須チェック・数値範囲など）。

|===

== バックエンド構成（Node.js + Express + TypeScript）

=== ディレクトリ構成

[source]
----
backend/
├── src/
│   ├── index.ts                 # サーバー起動・ミドルウェア登録
│   ├── middleware/
│   │   ├── auth.ts              # JWT 検証ミドルウェア（verifyToken）
│   │   └── errorHandler.ts      # 集中エラーハンドラー
│   ├── routes/
│   │   ├── auth.ts              # POST /api/auth/login, POST /api/auth/logout
│   │   ├── orders.ts            # CRUD /api/orders, /api/orders/:id
│   │   ├── customers.ts         # GET /api/customers
│   │   └── stats.ts             # GET /api/stats
│   ├── services/
│   │   ├── authService.ts       # bcrypt 検証・JWT 生成
│   │   ├── orderService.ts      # 注文 DB 操作
│   │   └── statsService.ts      # 統計集計 SQL
│   ├── db/
│   │   └── supabase.ts          # Supabase クライアント初期化
│   └── types/
│       └── index.ts             # 型定義・Express Request 拡張
----

=== 主要ミドルウェア・サービス設計

[cols="3,5", options="header"]
|===
|ファイル |担当責務

|`middleware/auth.ts`
|`Authorization: Bearer <JWT>` を検証し `req.user`（`{ sub, email }`）をセット。失敗時 401 を返す（認証設計書参照）。

|`middleware/errorHandler.ts`
|ルートハンドラーで発生した例外を一括捕捉し適切な HTTP ステータスと JSON エラーを返す。

|`services/authService.ts`
|`login(email, password)`: DB から users を参照し `bcrypt.compare` で認証。成功時 `jwt.sign` でトークンを生成して返す。

|`services/orderService.ts`
|`findAll(filter)` / `findById(id)` / `create(data)` / `update(id, data)` / `remove(id)` の CRUD を Supabase クライアント経由で実行。

|`services/statsService.ts`
|`getStats()`: orders テーブルを集計して総注文数・売上合計・ステータス別件数を返す。

|`db/supabase.ts`
|`@supabase/supabase-js` で Supabase クライアントを初期化・エクスポートする。接続情報は `DATABASE_URL` 環境変数から取得。

|===

== 主要機能一覧

[cols="1,3,4", options="header"]
|===
^.^|No. |機能名 |処理概要

^.^|1
|ログイン
|メール・パスワード認証。bcrypt 検証後 JWT を返す。フロントエンドで localStorage に保存する。

^.^|2
|ログアウト
|フロントエンドで localStorage からトークンを削除し `/login` へ遷移。

^.^|3
|ダッシュボード表示
|統計（総注文数・売上・ステータス別件数）と直近5件の注文を一覧表示する。

^.^|4
|注文一覧表示
|ステータスフィルター・ページネーション（20件/p）に対応。作成日降順がデフォルト。

^.^|5
|注文詳細表示
|注文ヘッダー + `order_items` の JOIN 結果を表示する。

^.^|6
|注文追加
|フォームから新規注文を入力し `POST /api/orders` で登録。バリデーションエラーはフィールド単位で表示。

^.^|7
|注文編集
|既存注文の全フィールドを編集できる。`PUT /api/orders/:id` で更新。

^.^|8
|注文削除
|確認ダイアログ表示後 `DELETE /api/orders/:id` で削除。注文明細も CASCADE 削除される。

^.^|9
|顧客一覧表示
|orders をメールで集計した顧客情報（名前・注文回数・累計金額）を表示する。

|===

== セキュリティ考慮事項

[cols="3,5", options="header"]
|===
|リスク |対策

|XSS
|React は JSX でのレンダリング時に自動エスケープを行う。`dangerouslySetInnerHTML` は使用しない。

|SQL インジェクション
|Supabase クライアントのプリペアドクエリを使用。生 SQL は `$1, $2` プレースホルダーを使う。

|認証バイパス
|`verifyToken` ミドルウェアを認証必須ルートに全適用。JWT の秘密鍵は環境変数で管理。

|ブルートフォース
|`express-rate-limit` を `/api/auth/login` に適用（最大 10 回/分）。

|CORS
|`cors` ミドルウェアで `CORS_ORIGIN`（Vercel の URL）のみ許可。

|===

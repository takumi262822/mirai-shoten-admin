== 各画面の詳細

=== 1. ログイン画面（/login）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`LoginPage`

|入力フィールド
|メールアドレス（必須）、パスワード（必須）

|バリデーション
|メール形式チェック・パスワード空白チェック（クライアント側）

|送信先
|`POST /api/auth/login`

|成功時の動作
|JWT を `localStorage.setItem("token", ...)` に保存 → `/dashboard` へリダイレクト

|失敗時の動作
|エラーメッセージをフォーム下部に表示（「メールアドレスまたはパスワードが正しくありません」）

|既ログイン時
|`/login` アクセス時に JWT が有効であれば `/dashboard` へリダイレクト

|===

=== 2. ダッシュボード（/dashboard）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`DashboardPage`

|統計カード
|総注文数 / 売上合計（¥表示） / 処理中件数 / 配達完了件数

|データ取得
|`GET /api/stats`

|最新注文
|`GET /api/orders?limit=5&sort=createdAt_desc`（5件のみ）

|更新タイミング
|画面表示時に毎回 API を呼び出す（キャッシュなし）

|===

=== 3. 注文一覧（/orders）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`OrderListPage`

|テーブルカラム
|注文ID / 顧客名 / メール / 数量 / 金額 / ステータス / 操作

|フィルター
|ステータス別（全・受付中・処理中・発送済み・配達完了・キャンセル）

|ページネーション
|1ページ 20件。`?page=&limit=` クエリパラメータで制御

|操作ボタン
|「詳細・編集」 → `/orders/:id` へ遷移 / 「削除」 → 確認ダイアログ後 `DELETE /api/orders/:id`

|データ取得
|`GET /api/orders?status=&page=&limit=`

|===

=== 4. 注文詳細・編集（/orders/:id）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`OrderDetailPage`

|表示内容
|注文全フィールド（顧客情報・住所・注文明細・ステータス・日時）

|編集項目
|顧客名・フリガナ・メール・電話番号・郵便番号・住所・ステータス

|更新送信先
|`PUT /api/orders/:id`

|削除
|`DELETE /api/orders/:id`（「本当に削除しますか？」確認ダイアログ表示後）

|削除後の動作
|`/orders` へリダイレクト

|データ取得
|`GET /api/orders/:id`

|===

=== 5. 注文追加（/orders/new）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`OrderNewPage`

|入力フィールド
|顧客名（必須）・フリガナ・メール（必須）・電話番号・郵便番号・住所・商品コード・数量（必須）・合計金額（必須）・ステータス

|バリデーション
|必須項目空白チェック / メール形式 / 数量は 1〜999 の整数 / 金額は 0 以上の整数

|送信先
|`POST /api/orders`

|成功後の動作
|`/orders` へリダイレクト

|失敗時の動作
|フィールド単位でエラーメッセージ表示（例: 「数量は1以上を入力してください」）

|===

=== 6. 顧客一覧（/customers）

[cols="2,5", options="header"]
|===
|項目 |内容

|コンポーネント
|`CustomerListPage`

|テーブルカラム
|顧客名 / メール / 注文回数 / 累計金額

|データ取得
|`GET /api/customers`（メールで GROUP BY した集計結果）

|===

== 共通仕様

[cols="2,5", options="header"]
|===
|項目 |内容

|未認証アクセス
|`/login` 以外へのアクセス時に JWT が無効・期限切れの場合は `/login` へリダイレクト

|ナビゲーション
|左サイドバー or 上部ヘッダーに「ダッシュボード / 注文一覧 / 注文追加 / 顧客一覧 / ログアウト」を表示

|ログアウト
|`localStorage.removeItem("token")` 後に `/login` へリダイレクト

|エラー画面
|存在しないルートには 404 コンポーネントを表示

|===

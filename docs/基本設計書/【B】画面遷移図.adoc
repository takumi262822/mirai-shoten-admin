== 遷移条件の補足

[cols="2,3,5", options="header"]
|===
|遷移元 |遷移先 |条件

|アプリ起動（`/`）
|`/login`
|localStorage に JWT がない、または JWT 検証が失敗した場合

|アプリ起動（`/`）
|`/dashboard`
|localStorage の JWT が有効な場合

|`/login`
|`/dashboard`
|`POST /api/auth/login` が `200 OK` を返した場合

|`/login`
|`/login`（変化なし）
|`POST /api/auth/login` が `401` を返した場合。エラーメッセージを表示

|任意の認証必要画面
|`/login`
|API から `401 Unauthorized` が返された場合（JWT 期限切れ等）。localStorage の token を削除してリダイレクト

|`/orders/new`
|`/orders`
|`POST /api/orders` が `201 Created` を返した場合

|`/orders/:id`
|`/orders`
|`DELETE /api/orders/:id` が `200 OK` を返した場合

|===

== ルート保護の実装方針

認証が必要なルートは React Router の `PrivateRoute` コンポーネントでラップする。

[source,typescript]
....
// PrivateRoute コンポーネントのイメージ
const PrivateRoute = ({ children }: { children: ReactNode }) => {
  const token = localStorage.getItem("token");
  // トークン未存在の場合はログイン画面にリダイレクトする
  if (!token) return <Navigate to="/login" replace />;
  return <>{children}</>;
};

// ルーティング設定
<Routes>
  <Route path="/login" element={<LoginPage />} />
  <Route path="/dashboard" element={<PrivateRoute><DashboardPage /></PrivateRoute>} />
  <Route path="/orders" element={<PrivateRoute><OrderListPage /></PrivateRoute>} />
  <Route path="/orders/new" element={<PrivateRoute><OrderNewPage /></PrivateRoute>} />
  <Route path="/orders/:id" element={<PrivateRoute><OrderDetailPage /></PrivateRoute>} />
  <Route path="/customers" element={<PrivateRoute><CustomerListPage /></PrivateRoute>} />
  <Route path="*" element={<NotFoundPage />} />
</Routes>
....

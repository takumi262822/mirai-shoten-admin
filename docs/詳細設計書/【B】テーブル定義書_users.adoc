== カラム定義

[cols="1,3,3,3,2,1,4", options="header", align="center"]
|===
^.^|No. ^.^|カラム名（論理名） ^.^|カラム名（物理名） ^.^|型 ^.^|制約 ^.^|PK ^.^|備考

^.^|1 <.^|ユーザーID <.^|`id` <.^|UUID <.^|PRIMARY KEY <.^|PK <.^|`gen_random_uuid()` で自動生成
^.^|2 <.^|メールアドレス <.^|`email` <.^|VARCHAR(255) <.^|UNIQUE INDEX, NOT NULL <.^|- <.^|`lower(email)` でユニークインデックスを貼り、大文字小文字を区別しない
^.^|3 <.^|パスワードハッシュ <.^|`password_hash` <.^|VARCHAR(255) <.^|NOT NULL <.^|- <.^|bcrypt (saltRounds=12) でハッシュ化。平文は保存しない
^.^|4 <.^|氏名 <.^|`name` <.^|VARCHAR(100) <.^|- <.^|- <.^|管理者の表示名
^.^|5 <.^|作成日時 <.^|`created_at` <.^|TIMESTAMPTZ <.^|NOT NULL <.^|- <.^|`DEFAULT NOW()`
^.^|6 <.^|更新日時 <.^|`updated_at` <.^|TIMESTAMPTZ <.^|NOT NULL <.^|- <.^|`DEFAULT NOW()`。UPDATE トリガーで自動更新
|===

== バリデーション制約

[cols="2,3,5", options="header", align="center"]
|===
^.^|カラム名 ^.^|制約種別 ^.^|内容

<.^|`email` <.^|形式 <.^|RFC 準拠の正規表現でチェック。255 文字以内
<.^|`password_hash` <.^|bcrypt 形式 <.^|`$2b$` で始まる 60 文字以上の文字列（登録時は平文を1〜72 文字で検証してからハッシュ化）
|===

== DDL

[source,sql]
----
CREATE TABLE users (
  id            UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  email         VARCHAR(255) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name          VARCHAR(100),
  created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
  updated_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- 大文字小文字を区別しないユニーク制約
CREATE UNIQUE INDEX users_email_unique ON users (lower(email));
----

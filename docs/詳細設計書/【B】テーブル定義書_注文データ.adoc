== カラム定義

[cols="1,3,3,3,2,1,4", options="header", align="center"]
|===
^.^|No. ^.^|カラム名（論理名） ^.^|カラム名（物理名） ^.^|型 ^.^|制約 ^.^|PK ^.^|備考

^.^|1 <.^|注文ID <.^|`id` <.^|UUID <.^|PRIMARY KEY <.^|PK <.^|`gen_random_uuid()` で自動生成
^.^|2 <.^|顧客名 <.^|`customer_name` <.^|VARCHAR(100) <.^|NOT NULL <.^|- <.^|氏名（フルネーム）
^.^|3 <.^|顧客名（フリガナ） <.^|`customer_kana` <.^|VARCHAR(100) <.^|- <.^|- <.^|カタカナ表記
^.^|4 <.^|メールアドレス <.^|`email` <.^|VARCHAR(255) <.^|NOT NULL <.^|- <.^|顧客連絡先
^.^|5 <.^|電話番号 <.^|`tel` <.^|VARCHAR(20) <.^|- <.^|- <.^|例: `090-1234-5678`
^.^|6 <.^|郵便番号 <.^|`zip` <.^|VARCHAR(10) <.^|- <.^|- <.^|例: `123-4567`
^.^|7 <.^|住所 <.^|`address` <.^|TEXT <.^|- <.^|- <.^|都道府県・市区町村・番地を含む全住所
^.^|8 <.^|合計金額 <.^|`total_price` <.^|INTEGER <.^|NOT NULL, CHECK >= 0 <.^|- <.^|円単位。0 以上の整数
^.^|9 <.^|注文ステータス <.^|`status` <.^|VARCHAR(20) <.^|NOT NULL, CHECK <.^|- <.^|下記「ステータス定義」参照。DEFAULT: `pending`
^.^|10 <.^|作成日時 <.^|`created_at` <.^|TIMESTAMPTZ <.^|NOT NULL <.^|- <.^|`DEFAULT NOW()`
^.^|11 <.^|更新日時 <.^|`updated_at` <.^|TIMESTAMPTZ <.^|NOT NULL <.^|- <.^|`DEFAULT NOW()`。UPDATE トリガーで自動更新
|===

== ステータス定義

[cols="2,3,4", options="header", align="center"]
|===
^.^|コード（物理値） ^.^|表示名（論理名） ^.^|説明

<.^|`pending` <.^|受付中 <.^|注文受付直後の初期ステータス
<.^|`processing` <.^|処理中 <.^|出荷準備・商品ピッキング中
<.^|`shipped` <.^|発送済み <.^|集荷済みで配送中
<.^|`delivered` <.^|配達完了 <.^|お客様へ届いた状態
<.^|`cancelled` <.^|キャンセル <.^|注文取消し済み
|===

== バリデーション制約

[cols="2,3,5", options="header", align="center"]
|===
^.^|カラム名 ^.^|制約種別 ^.^|内容

<.^|`customer_name` <.^|必須・長さ <.^|1〜100 文字。空文字不可
<.^|`email` <.^|形式 <.^|RFC 準拠の正規表現でチェック
<.^|`total_price` <.^|範囲 <.^|0 以上の整数
<.^|`status` <.^|列挙値 <.^|5 値のいずれか（上記ステータス定義参照）
|===

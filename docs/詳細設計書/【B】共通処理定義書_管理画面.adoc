== 概要

本書は未来商店管理画面（mirai-shoten-admin）全体で使用する共通ユーティリティの仕様を記載します。 +
XSS 対策・入力バリデーション・UI コンポーネントを対象とします。

---

== 1. XSSProtectionAdmin クラス

=== 格納先

`src/utils/xss.js`

=== 説明

管理画面へのユーザー入力をエスケープ・サニタイズし、XSS 攻撃を防ぐ。 +
全角英数字の半角正規化とメールアドレス・数値範囲の検証も担う。

=== 静的メソッド

==== escape(str)

文字列内の HTML 特殊文字をエンティティに置換して返す。

[cols="2,2,4", options="header"]
|===
^.^|項目 ^.^|型 <.^|内容

<.^|引数 `str` |any |任意の値。非文字列の場合は空文字を返す。
<.^|戻り値 |string |HTML 特殊文字をエンティティに置換した文字列
|===

.変換表
[cols="1,1", options="header"]
|===
|入力文字|変換後エンティティ
|`&`|`&amp;`
|`<`|`&lt;`
|`>`|`&gt;`
|`"`|`&quot;`
|`'`|`&#x27;`
|===

*実装方針:*

- 各文字を個別に `replace` で置換する（チェーン方式）
- 非文字列は即座に空文字を返す

---

==== sanitizeInput(input)

入力値に対してトリム→危険パターン検出→エスケープの順で安全化する。

[cols="2,2,4", options="header"]
|===
^.^|項目 ^.^|型 <.^|内容

<.^|引数 `input` |any |任意の値。`null` / 非文字列の場合は空文字を返す。
<.^|戻り値 |string |サニタイズ済み文字列（危険な入力は空文字）
|===

*処理フロー:*

. `null` または非文字列の場合は空文字を返す
. `input.trim()` で前後空白を除去する
. 以下のいずれかが検出された場合は空文字を返す
  * `<script>...</script>` タグパターン
  * `javascript:` プロトコル
  * `onXxx=` 形式のインラインイベントハンドラ
. 検出されなかった場合は `escape(normalized)` を呼び出して返す

---

==== normalizeFullWidthAscii(str)

全角英数字・スペースを半角に変換して返す。

[cols="2,2,4", options="header"]
|===
^.^|項目 ^.^|型 <.^|内容

<.^|引数 `str` |any |任意の文字列。falsy または非文字列の場合は空文字を返す。
<.^|戻り値 |string |半角変換済み文字列
|===

.変換ルール
[cols="2,2", options="header"]
|===
|変換前|変換後
|全角英大文字 `Ａ-Ｚ`|半角英大文字 `A-Z`
|全角英小文字 `ａ-ｚ`|半角英小文字 `a-z`
|全角数字 `０-９`|半角数字 `0-9`
|全角スペース `　`（U+3000）|半角スペース ` `
|===

*実装方針:*

- 文字コード差分 `0xFEE0` を利用して全角→半角変換する
- 全角スペースは `replace(/　/g, ' ')` で置換する

---

==== isValidEmail(email)

メールアドレスの形式を簡易検証する。

[cols="2,2,4", options="header"]
|===
^.^|項目 ^.^|型 <.^|内容

<.^|引数 `email` |any |任意の値。falsy または非文字列の場合は `false` を返す。
<.^|戻り値 |boolean |`/^[^\s@]+@[^\s@]+\.[^\s@]+$/` に一致すれば `true`
|===

---

==== isInRange(value, min, max)

数値が指定した範囲内にあるかを検証する。

[cols="2,2,4", options="header"]
|===
^.^|項目 ^.^|型 <.^|内容

<.^|引数 `value` |number\|string |検査対象値（`Number()` で変換して評価する）
<.^|引数 `min` |number |最小値（含む）
<.^|引数 `max` |number |最大値（含む）
<.^|戻り値 |boolean |`Number(value)` が NaN でなく `min <= n <= max` であれば `true`
|===

=== 主要メソッド

[cols="2,4", options="header"]

|===

== 4. 使用上の共通ルール

. 管理画面に表示する全文字列は `XSSProtectionAdmin.escape()` または `sanitizeInput()` を通す
. 検索フォームの入力値は `sanitizeInput()` でサニタイズしてから処理する
. 全角入力はフォーム送信時に `normalizeFullWidthAscii()` で正規化する
. localStorage の直接操作は `AdminManager.loadOrders()` / `updateStatus()` 経由に統一する

== 採用方式

[cols="2,4", options="header"]
|===
|項目 |内容

|認証方式
|JWT (JSON Web Token) — HS256 署名

|パスワードハッシュ
|bcrypt（saltRounds: 12）

|トークン保存場所
|フロントエンドの `localStorage` に保存

|トークン送信方法
|`Authorization: Bearer <JWT>` ヘッダー

|トークン有効期限
|8 時間（例: `JWT_EXPIRES_IN=8h`）

|ログアウト
|フロントエンドで localStorage からトークンを削除（サーバー側はブラックリスト管理なし）

|===

== API 認証ミドルウェア（verifyToken）

認証が必要なすべてのエンドポイントはこのミドルウェアを通過する。

[source,typescript]
....
// backend/src/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';

export type AuthPayload = { sub: string; email: string; iat: number; exp: number };

export function verifyToken(req: Request, res: Response, next: NextFunction): void {
  const authHeader = req.headers.authorization;

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    res.status(401).json({ error: '認証が必要です' });
    return;
  }

  const token = authHeader.slice(7);

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET!) as AuthPayload;
    req.user = payload;  // Request 型を拡張して格納
    next();
  } catch (err) {
    if (err instanceof jwt.TokenExpiredError) {
      res.status(401).json({ error: 'トークンの有効期限が切れています' });
    } else {
      res.status(401).json({ error: '無効なトークンです' });
    }
  }
}
....

ミドルウェアの適用例:

[source,typescript]
....
// backend/src/routes/orders.ts
import { Router } from 'express';
import { verifyToken } from '../middleware/auth';

const router = Router();

router.use(verifyToken);  // このルーター配下のすべてに認証を要求

router.get('/', getOrders);
router.post('/', createOrder);
...

export default router;

== フロントエンドのトークン管理

[source,typescript]
....
// frontend/src/utils/api.ts
const API_BASE = import.meta.env.VITE_API_URL;

export async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const token = localStorage.getItem('token');
  const res = await fetch(`${API_BASE}${path}`, {
    ...init,
    headers: {
      'Content-Type': 'application/json',
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
      ...init?.headers,
    },
  });

  if (res.status === 401) {
    // トークン期限切れ → ログイン画面へ
    localStorage.removeItem('token');
    window.location.href = '/login';
    throw new Error('認証切れ');
  }

  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw new Error(body.error ?? `HTTP ${res.status}`);
  }

  return res.json() as Promise<T>;
}
....

== PrivateRoute（未認証リダイレクト）

[source,typescript]
....
// frontend/src/components/PrivateRoute.tsx
import { Navigate, Outlet } from 'react-router-dom';

export function PrivateRoute() {
  const token = localStorage.getItem('token');
  return token ? <Outlet /> : <Navigate to="/login" replace />;
}
....

== セキュリティ考慮事項

[cols="3,5", options="header"]
|===
|リスク |対策

|パスワード漏洩
|bcrypt (saltRounds=12) でハッシュ化。平文パスワードはログにも残さない。

|ブルートフォース攻撃
|bcrypt の計算コストに加え、Express レートリミット（express-rate-limit）を /api/auth/login に適用する。

|JWT 秘密鍵漏洩
|`JWT_SECRET` は環境変数で管理。Render のシークレット管理機能を利用。32 文字以上のランダム文字列を使用する。

|XSS によるトークン窃取
|フロントエンドで入力値を DOMPurify でサニタイズ。API レスポンスを innerHTML に直接埋め込まない。

|CORS
|Express の `cors` ミドルウェアで `CORS_ORIGIN`（Vercel の URL）のみ許可。

|SQL インジェクション
|Supabase クライアントのプリペアドステートメントを使用。生 SQL は `$1, $2` プレースホルダーを使う。

|===

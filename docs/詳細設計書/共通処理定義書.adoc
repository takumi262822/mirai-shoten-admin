== 概要

本書は MIRAI SHOTEN 管理画面（mirai-shoten-admin）全体で使用する共通ユーティリティの仕様を記載します。
XSS対策・入力バリデーション・UIコンポーネントを対象とします。

---

== 1. XSSProtectionAdmin クラス

=== 所在

`src/utils/xss.js`

=== 責務

管理画面へのユーザー入力をエスケープ・サニタイズし、XSS 攻撃を防ぐ。
全角英数字の半角正規化とメールアドレス・数値範囲の検証も担う。

=== 静的メソッド

==== escape(str)

文字列内の HTML 特殊文字をエンティティに置換して返す。

[cols="1,3", options="header"]
|===
|引数
|説明

|`str`
|任意の値。非文字列の場合は空文字を返す。
|===

*変換表:*

[cols="1,1", options="header"]
|===
|入力文字
|変換後エンティティ

|`&`
|`&amp;`

|`<`
|`&lt;`

|`>`
|`&gt;`

|`"`
|`&quot;`

|`'`
|`&#x27;`
|===

*実装方針:*
- 各文字を個別に `replace` で置換する（チェーン方式）
- 非文字列は即座に空文字を返す

---

==== sanitizeInput(input)

入力値に対してトリム→危険パターン検出→エスケープの順で安全化する。

[cols="1,3", options="header"]
|===
|引数
|説明

|`input`
|任意の値。`null` / 非文字列の場合は空文字を返す。
|===

*処理フロー:*

. `null` または非文字列の場合は空文字を返す
. `input.trim()` で前後空白を除去する
. 以下のいずれかが検出された場合は空文字を返す
  * `<script>...</script>` タグパターン
  * `javascript:` プロトコル
  * `onXxx=` 形式のインラインイベントハンドラ
. 検出されなかった場合は `escape(normalized)` を呼び出して返す

*戻り値:* サニタイズ済み文字列（危険な入力は空文字）

---

==== normalizeFullWidthAscii(str)

全角英数字・スペースを半角に変換して返す。

[cols="1,3", options="header"]
|===
|引数
|説明

|`str`
|任意の文字列。falsy または非文字列の場合は空文字を返す。
|===

*変換ルール:*

[cols="2,2", options="header"]
|===
|変換前
|変換後

|全角英大文字 `Ａ-Ｚ`
|半角英大文字 `A-Z`

|全角英小文字 `ａ-ｚ`
|半角英小文字 `a-z`

|全角数字 `０-９`
|半角数字 `0-9`

|全角スペース `　`（U+3000）
|半角スペース ` `
|===

*実装方針:*
- 文字コード差分 `0xFEE0` を利用して全角→半角変換する
- 全角スペースは `replace(/　/g, ' ')` で置換する

---

==== isValidEmail(email)

メールアドレスの形式を簡易検証する。

[cols="1,3", options="header"]
|===
|引数
|説明

|`email`
|任意の値。falsy または非文字列の場合は `false` を返す。
|===

*判定条件:* `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` に一致すること

*戻り値:* `true` / `false`

---

==== isInRange(value, min, max)

数値が指定した範囲内にあるかを検証する。

[cols="1,1,3", options="header"]
|===
|引数
|型
|説明

|`value`
|`number\|string`
|検査対象値（`Number()` で変換して評価する）

|`min`
|`number`
|最小値（含む）

|`max`
|`number`
|最大値（含む）
|===

*判定条件:*

. `Number(value)` が `NaN` でないこと
. `num >= min && num <= max` であること

*戻り値:* `true` / `false`

== 3. AdminManager クラス

=== 所在

`src/core/admin-manager.js`

=== 責務

localStorage の `adminOrders` キーから注文データを読み込み、フィルタリング・ステータス更新・集計を行う。

[cols="1,3", options="header"]
|===
|メソッド
|説明

|`loadOrders()`
|localStorage から注文データを読み込み、旧形式を正規化して返す

|`filterOrders(orders, filters)`
|ステータス・日付・キーワードで絞り込んだ注文配列を返す

|`updateStatus(orderId, newStatus)`
|指定注文のステータスを更新して localStorage に保存する

|`calcSummary(orders)`
|注文件数・合計金額・ステータス別件数を集計して返す
|===

---

== 4. 使用上の共通ルール

. 管理画面に表示する全文字列は `XSSProtectionAdmin.escape()` または `sanitizeInput()` を通す
. 検索フォームの入力値は `sanitizeInput()` でサニタイズしてから処理する
. 全角入力はフォーム送信時に `normalizeFullWidthAscii()` で正規化する
. localStorage の直接操作は `AdminManager.loadOrders()` / `updateStatus()` 経由に統一する
